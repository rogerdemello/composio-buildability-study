04

The agent that did it

A pipeline built on Composio's own SDK + tools. Humans set the rubric and adjudicated the edges.

Pipeline

1

Research agent · research_agent.py

Per app, a Claude loop calls Composio's own COMPOSIO_SEARCH tools to find and read the official developer docs, then a forced tool-call emits one structured record against a fixed schema. 10 category workers ran in parallel.

2

Consolidate + cluster · analyze.py

100 rows validated against the schema, then clustered into the auth / access / buildability / MCP patterns you see above.

3

Verification loop · verify.py

A different, stronger model with an adversarial "assume this is wrong" prompt re-researched a stratified sample from scratch and diffed it. Every evidence URL was fetched for a live 200.

4

Human adjudication

Disagreements were flagged, never auto-overwritten. A human read the disputed docs and corrected the master dataset.

⚠ Where a human was needed

  • Login-gated / thin docs — fanbasis, iPayX, Paygent, higgsfield: the agent can't read behind a login wall, so it correctly downgraded confidence and a human made the call.
  • Sandbox vs production — "self-serve sandbox, KYC-gated production" (Plaid, Ramp, Binance) is a judgment the rubric couldn't fully encode.
  • Composio catalog stubs — a toolkit page with 0 tools is not real coverage; a human distinguished "listed" from "usable".
  • Blocked vs moderate — on enterprise apps a partner path may exist but isn't self-serve; the line is a human call.

Run it yourself

The agent is the deliverable. Everything here regenerates from the repo.
$ reproduce the study
# 1. research all 100 apps with the agent
python -m agent.research_agent
# 2. consolidate + find the patterns
python -m agent.analyze consolidate
python -m agent.analyze patterns
# 3. audit a sample + grade accuracy
python -m agent.verify --sample 20

Honesty notes

  • No paid accounts were used. Where an app is gated behind payment or partnership, that is the finding — reported with evidence, not treated as a failure.
  • Confidence is per-row and visible in the table (green/amber/red dot). Low-confidence rows are the obscure, login-walled ones — called out, not hidden.
  • Verification covers a 20-app stratified sample, not all 100. The accuracy number is the sample's; the full set is estimated to track it within a few points.
  • One app genuinely resisted the agent: Paygent Connect — Japanese, contract-only, mutual-TLS; the "NMI-powered" hint appears to be wrong. We report that rather than guess.
