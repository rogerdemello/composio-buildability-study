    04

    The agent that did it

    A modular pipeline on Composio's own SDK + tools. Humans set the rubric and adjudicated the edges.

    Pipeline

    P

    Providers

    Composio search → Tavily → Firecrawl → plain fetch, in preference order. Playwright browser is a fallback, invoked only when a page is JS-walled or the verifier flags an inconsistency.

    1

    Researcher · agents/researcher.py

    Gathers docs, then forces a structured record where every field carries its own value + confidence + evidence URL. Runs async under a Semaphore(5). For this submission the same structure ran as Claude Code web-search sub-agents across the 10 categories — see the honesty note below.

    2

    Verifier · agents/verifier.py

    A different, stronger model with an adversarial prompt re-researches a sample and diffs it; every evidence URL is fetched for a live 200. Sets each field's PASS/FAIL/UNKNOWN.

    3

    Pattern engine · agents/pattern_analyzer.py

    Feeds the whole dataset to an LLM for the insights up top — grounded in code-computed counts so it can't invent a number.

    4

    Report · agents/report_writer.py

    Renders this page straight from JSON. Nothing here is hand-written.

    ⚠ Where a human was needed

    • Login-gated / thin docs — fanbasis, iPayX, Paygent, higgsfield: the agent can't read behind a login wall, so it downgraded confidence and a human made the call.
    • Sandbox vs production — "self-serve sandbox, KYC-gated production" (Plaid, Ramp, Binance) needed a human to encode as sandbox-self-serve.
    • Composio catalog stubs — a toolkit page with 0 tools is not real coverage; a human distinguished "listed" from "usable".
    • Access reclassification — the score exposed apps the agent hedged as "moderate" that were really gated (SFCC, Brex) or really self-serve; a human set the access value the score keys off.

    ⚑ Honesty — what actually ran on Composio

    The pipeline in the repo (providers/composio_search.py + agents/researcher.py) is built to research each app with Composio's own COMPOSIO_SEARCH tools via the Composio SDK/MCP — that code is committed and runnable. For this submission the 100-app dataset was produced by the same agent structure running inside Claude Code (WebSearch/WebFetch sub-agents: 10 category researchers + 4 independent auditors) against the identical schema — not by the Composio path. The Composio MCP server is registered; a live Composio-powered run is one auth step away (AuthKit OAuth, or a Composio project API key for the SDK path). We flag this rather than imply Composio produced the numbers.

    05

    How it was built

    Perfect the pipeline on 5, then scale to 100. The score turns "easy" into a defensible number.
    Phase 1

    5 golden apps

    Salesforce, HubSpot, Slack, Stripe, Shopify — five auth models across five categories. The pipeline was tuned until these came out clean and consistent.

    → then scaled to 100
    Phase 2

    Async fan-out

    All 100 researched concurrently under a Semaphore(5) for rate-limiting; resumable via a JSON store so a crash never re-does finished work.

    → minutes, not hours
    Phase 3

    Score, don't guess

    Buildability is a weighted 0–100 score (auth, surface, breadth, MCP, access) with gated access hard-capped. The verdict is the score band — click any Score cell to see the maths.

    → avg /100
    Phase 4

    Verify + lift

    An independent adversarial audit + evidence-liveness on a stratified sample. It caught real errors and moved accuracy up — shown honestly below.

    → accuracy

    What defeated the agent

    If an app beat us, it's here — not hidden.
    Reading the table. access: self-serve = mint creds yourself free/trial · sandbox-self-serve = free sandbox, gated production · gated-approval = app-review/dev-token/KYC · gated-paid = needs a paid plan · gated-partner = contact-sales/enterprise · trial-only = time-limited trial.

    08

    Run it yourself

    The agent is the deliverable. This whole page regenerates from JSON.
    $ reproduce the study
    # perfect the pipeline on the 5 golden apps first
python main.py research --golden
# then scale to all 100 (async, Semaphore(5), resumable)
python main.py research
# independent adversarial audit of a stratified sample
python main.py verify --sample 20
# LLM pattern engine + render this page from JSON
python main.py patterns
python main.py report
# …or the whole chain:
python main.py all

    Honesty notes

    • No paid accounts were used. Where an app is gated behind payment or partnership, that is the finding — reported with evidence, not treated as a failure.
    • Confidence is per field, not per app, and visible in every provenance popup. Lower-confidence values are dotted amber in the table.
    • Verification independently graded 80 input fields on a 20-app stratified sample, not all 100. The accuracy number is the sample's.
    • Only of 100 apps are human-checked (the audited sample); the rest carry the agent's own confidence. That distinction is shown, not blurred.
